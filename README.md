## Что даёт мод

FrogHelper собирает важную игровую информацию в одном месте и показывает её прямо во время игры:

- таймеры боссов, способностей, посохов, зелий, продавцов и майнеров, а также состояния рун и питомцев;
- прогресс уровня и комбо;
- удобные HUD-виджеты с редактором расположения и режимом UnClutter;
- переключение наборов рун по настраиваемым хоткеям;
- карта данжа/осады, рецепты, fishing-информация, pop-up уведомления;
- улучшения чата и внутриигровой обмен данными между игроками с FrogHelper;
- Quick Access для быстрых команд по удержанию одной клавиши;
- ESP и подсветка полезных объектов;
- Discord Rich Presence с игровым контекстом;
- локальные уведомления о важных событиях.

## Быстрый старт

После входа в игру полезно знать несколько основных действий:

| Действие | По умолчанию |
| --- | --- |
| Включить режим редактирования HUD | `H` |
| Открыть меню настроек | `O` |
| Открыть Quick Access | удерживать `R` |
| Включить / выключить авто-кликер | `K` |

Дополнительно:

| Действие | По умолчанию |
| --- | --- |
| `clanhide` | `Z` |
| `runesbag` | `X` |

## Основные хоткеи

### `H` — Editing Mode

Открывает режим редактирования HUD. В нём можно:

- двигать виджеты;
- менять масштаб;
- привязывать их к якорям экрана;
- сцеплять виджеты между собой;
- таскать связанные группы;
- временно скрывать виджеты из обычной игры через кнопку "свернуть" на самом виджете.

### `O` — Settings Menu

Открывает внутреннее меню настроек FrogHelper. Через него настраиваются:

- виджеты;
- Quick Access;
- чат;
- pop-ups;
- fishing;
- render и highlight-опции;
- Discord RPC;
- темы;
- clicker и другие утилиты.

### `R` — Quick Access

Работает по принципу удержания:

1. зажимаете `R`;
2. наводитесь на нужную команду;
3. отпускаете клавишу;
4. команда выполняется автоматически.

### `K` — Clicker

Переключает авто-кликер. Он может:

- бить как обычная атака;
- использовать предмет вместо атаки;
- работать с заданным CPS.

### `Z` — Clan Hide

Отправляет команду `clanhide`.

### `X` — Runes Bag

Отправляет команду `runesbag`.

### `1`–`7` — выбор набора рун

При открытом окне рун-мешка эти клавиши переключают активный набор рун (1–7). Также работают стрелки `←` / `→` для перебора наборов.

Клавиши переназначаемые: `Options → Controls → категория FrogHelper → Rune Set 1…7`. По умолчанию — цифры `1`–`7`. Они срабатывают только внутри окна рун-мешка, поэтому не конфликтуют с выбором слотов хотбара в обычной игре.

## Редактор HUD

Редактор HUD нужен для полной подгонки интерфейса под себя.

### Что можно делать

- свободно двигать виджеты;
- привязывать их к экранным якорям;
- сцеплять один виджет с другим;
- переносить целые группы связанных элементов;
- менять масштаб каждого блока;
- сохранять персональную раскладку;
- скрывать отдельные виджеты только в обычной игре, не теряя их preview в редакторе.

### Управление в редакторе

| Действие | Как работает |
| --- | --- |
| Перемещение | зажать ЛКМ по виджету |
| Масштабирование | `Ctrl` + колесо мыши |
| Тонкая регулировка масштаба | `Ctrl` + `Shift` + колесо |
| Перетаскивание группы | удерживать `Alt` при начале перетаскивания |
| Привязка к якорю | поднести виджет к якорной точке |
| Скрыть только в игре | нажать на полупрозрачную кнопку "свернуть" на виджете |
| Сохранение позиции | автоматически после отпускания |

### Как работает скрытие виджетов

У виджетов есть два разных режима отключения:

- `Active` в настройках: полностью выключает виджет, включая preview в редакторе;
- кнопка "свернуть" в редакторе: скрывает виджет только на обычном игровом экране, но оставляет preview в editing mode.

## HUD и виджеты

Ниже перечислены основные HUD-элементы, которые мод умеет показывать игроку.

### Боссы и бой

| Виджет | Что показывает |
| --- | --- |
| **Boss Timers** | список боссов с таймерами возрождения |
| **Boss Bar** | кастомный блок с полосой босса |
| **Boss Damage** | ваш текущий или последний урон по боссу |
| **Ability Cooldowns** | откаты способностей |
| **Wand Cooldowns** | откаты палок / связанных умений |
| **Combo Progress** | текущий прогресс комбо |
| **Blocks/s** | скорость ломания блоков |
| **Estimated TPS** | клиентская оценка TPS сервера |

### Boss Timers

Список боссов с обратным отсчётом до возрождения. Дополнительно:

- уже возродившийся босс какое-то время продолжает висеть с таймером «в минусе» и подсвечивается цветом Hard Accent (см. [Тему](#тема-и-внешний-вид)), а не пропадает мгновенно;
- диапазон уровней (`Min level` / `Max level`) настраивается; верхняя граница автоматически расширяется, когда сервер вводит боссов выше текущего потолка;
- источник данных выбирается между протоколом и миром (`Source mode`), плюс есть настройки иконок, имён, уровней и выравнивания.

### Wand Cooldowns

Таймеры посохов приходят из серверного протокола. Для Посоха Ветра, Посоха Вихря и локального названия `Величие гарпии` использование дополнительно отслеживается по предмету в руке, чтобы откат появлялся без ожидания следующего пакета.

Тумблер `Numeric Cooldown` заменяет полосу прогресса числом. В последнюю секунду оставшееся время показывается с десятыми долями секунды.

### Estimated TPS

Виджет показывает:

- текущий `TPS`;
- `1% low`;
- `0.1% low`.

Для него есть отдельный флаг `Enable Estimated TPS Monitor`. Если он выключен, мод вообще не собирает сэмплы и не тратит ресурсы на расчёт.

### Прогресс и активные эффекты

| Виджет | Что показывает |
| --- | --- |
| **Level Progress** | прогресс уровня |
| **Boosters** | активные бустеры и их длительность |
| **Potions** | откат до следующего доступного приёма зелья и период готовности после него |
| **Active Runes** | активные руны |
| **Active Pets** | активных питомцев |
| **Miners** | статус майнеров |
| **Seller Cooldowns** | откаты продавцов и похожих систем |
| **Visibility Status** | текущий режим видимости игроков / клана / питомцев |

### Рецепты и полезные панели

| Виджет | Что делает |
| --- | --- |
| **Potion Recipe** | показывает рецепт текущего зелья |
| **Craft Recipe** | показывает рецепт крафта |
| **Dungeon / Siege Map** | отображает карту данжа / осады |
| **Fishing Nibbles** | показывает информацию по клёву |
| **Fishing Quests** | показывает рыболовные задания с разделением по типу рыбы |
| **Entity Inspect** | блок с информацией о сущности под прицелом |
| **Scoreboard** | отдельный HUD-вариант scoreboard |
| **Chat Queue** | очередь отложенных исходящих сообщений |
| **Protocol Graph** | диагностическую статистику входящих протокольных данных |
| **Pop-Ups** | окно локальных всплывающих уведомлений |

### Дополнительно по виджетам

- большинство виджетов можно свободно двигать и масштабировать;
- layout сохраняется между запусками;
- виджеты можно сцеплять и перемещать группами;
- для части виджетов есть compact-режим;
- для тяжёлых функций добавлены предупреждения о влиянии на производительность.

### UnClutter

Тумблер `UnClutter (hide widget titles)` (категория `Widget`, включён по умолчанию) убирает подписи-названия внутри виджетов — остаются только сами данные (например, у `Active Runes` пропадает заголовок, остаются строки рун). Скрытие действует и в игре, и в редакторе, а фон и акцентная линия виджета остаются на месте. Виджеты, где верхняя строка сама по себе является данными (например `Level Progress`, `Combo Progress`), тумблер не затрагивает.

## Discord RPC

В FrogHelper встроен собственный Discord IPC-коннектор без внешних RPC-библиотек.

### Что показывает RPC

Discord Rich Presence умеет отображать:

- текущий сервер и зеркало;
- локацию;
- уровень;
- комбо;
- активный game event;
- elapsed time;
- режим активности по типу локации.

### Режимы активности

RPC автоматически подбирает статус и иконку в зависимости от ситуации:

| Режим | Что обычно означает |
| --- | --- |
| `froghelper` | нейтральный fallback |
| `hub` | хаб / спавн / лобби |
| `mine` | шахта |
| `dungeon` | данж |
| `siege` | осада |
| `boss` | босс |
| `fishing` | рыбалка |
| `death` | смерть / ожидание респавна |
| `event` | маленькая иконка активного ивента |

### Privacy Mode

Есть отдельный `Privacy mode`.

В нём RPC почти пустой:

- базовая иконка `froghelper`;
- строка `FrogHelper`;
- без сервера, локации, событий и прочих деталей.

### Настройки Discord

Во вкладке `Discord` можно управлять:

- `Enable auto RPC`
- `Privacy mode`
- `Show elapsed time`
- `Show server`
- `Show mirror`
- `Show location`
- `Show level`
- `Show combo`
- `Show game event`

## Чат и общение

### Вкладки чата

Сообщения автоматически раскладываются по вкладкам:

- общий чат;
- приваты;
- отдельные категории по префиксам и типам сообщений.

### Метки времени

Каждое сообщение можно снабжать временем в формате `HH:mm:ss`, чтобы было легче понимать, когда именно произошло событие.

### Tone Down Chat

Опция делает чат визуально спокойнее:

- убирает часть визуального шума;
- уменьшает агрессивную декоративность;
- делает переписку аккуратнее и легче читаемой.

### Копирование сообщений

ПКМ по строке чата копирует тело сообщения без timestamp, канального префикса, ранга и ника отправителя. Поведение включается тумблером `Copy chat by RMB`.

При включённом `Full Message Copy` копируется полная отображаемая строка: timestamp, канал, ранг, ник и текст. Внутренние служебные символы FrogHelper в буфер обмена не попадают.

После копирования можно получить pop-up уведомление.

### Auto Thanks

Если кто-то активировал глобальный бустер, мод может автоматически отправить `thx`.

### Приваты и pop-up уведомления

На входящий приват можно получить отдельный pop-up:

- с превью текста;
- без необходимости постоянно держать чат открытым.

### Обмен таймерами боссов

Команда `/fhshare <ник>` отправляет ваши локальные таймеры боссов другому игроку с FrogHelper.

### Пользователи FrogHelper и Socials

FrogHelper распознаёт других пользователей мода и может показывать рядом с ними значок жабки в чате и nametag. Отображение управляется настройкой `FrogHelper Badge On Nametags`.

Тумблер `Socials` управляет только исходящим участием в обнаружении и синхронизации:

- включён — к обычным сообщениям добавляется служебный маячок, а между пользователями выполняется фоновая синхронизация;
- выключен — маячок и исходящие пакеты синхронизации не отправляются, но входящие пакеты принимаются, а значки других пользователей продолжают отображаться.

## Quick Access

Quick Access — это быстрое секционное меню действий, открываемое удержанием клавиши.

### Что это даёт

- быстрый доступ к частым командам;
- меньше ручного ввода;
- удобный выбор действий буквально за одно удержание клавиши.

### Как это работает

1. зажимаете `R`;
2. на экране появляется меню с секциями и слотами;
3. наводитесь на нужное действие;
4. отпускаете `R`;
5. команда выполняется автоматически.

### Стандартные секции

По умолчанию есть заготовки вроде:

- `Common`
- `Travel`
- `Player`

### Подстановка `{player}`

Некоторые команды могут автоматически подставлять ник игрока под прицелом.

Это полезно для:

- статистики;
- трейда;
- быстрых действий по выбранной цели.

### Встроенный редактор Quick Access

Через настройки можно открыть отдельный редактор и:

- создавать секции;
- добавлять и удалять слоты;
- менять названия;
- менять команду;
- менять иконку предмета;
- задавать `CustomModelData`;
- перетаскивать элементы между секциями.

## Оверлеи и подсветка

### Fishing

Рыболовный блок умеет:

- показывать рыболовные метки в мире;
- выводить виджет по клёву;
- использовать данные протокола по fishing spots;
- автоматически дёргать удочку при срабатывании, если включён Auto Fish.

### Алхимия

Подсвечивает ингредиенты алхимии прямо в мире.

### Данжи и осады

Поддерживаются:

- карта данжа / осады;
- подсветка важных декоративных элементов и ориентиров.

### Полосы HP игроков

Мод умеет рисовать полосы здоровья над игроками:

- с настройкой прозрачности;
- с вертикальным смещением;
- с плавным затуханием по дистанции;
- с настройкой размера самой полоски (`Bar size %`).

Когда здоровье игрока опускается ниже настраиваемого порога (`Hard accent threshold %`), полоса уходит «в красное» — смещается к цвету Hard Accent. Насколько резко происходит смена цвета, задаётся ползунком `Accent strength %`: при высоком значении переход читается чётко, а не еле заметным затуханием.

### Useful Items Highlight

Подсветка полезных объектов и предметов помогает быстрее замечать важные точки в мире.

## Автоматизация и утилиты

### Auto Messages

Система авто-сообщений позволяет настроить слоты с сообщениями, которые будут отправляться по таймеру.

Для каждого слота задаются:

- текст;
- активность;
- задержка в секундах.

### Clicker

Позволяет включить авто-нажатие с заданной скоростью.

Настройки:

- CPS;
- режим атаки;
- режим использования предмета вместо атаки.

### Auto Fish

Полуавтоматическая рыбалка:

- отслеживает состояние поплавка;
- ждёт нужное количество тиков;
- автоматически делает нужное действие.

### Boss Respawn Messages

Мод умеет локально уведомлять о важных этапах босса:

- сообщение перед респавном;
- сообщение о спавне;
- сообщение о проклятии;
- сообщение о низком HP.

### Pop-Ups

Встроенная система всплывающих уведомлений сообщает о событиях вроде:

- входящего привата;
- появления босса;
- готовности способности;
- готовности wand-умения;
- готовности продавца;
- возвращения майнера;
- окончания отката набора рун;
- окончания зелья;
- окончания бустера;
- копирования чата.

Дополнительно настраиваются:

- число одновременно видимых pop-ups;
- длительность показа;
- скорость появления;
- скорость исчезновения.

## Тема и внешний вид

Мод поддерживает несколько пресетов темы интерфейса:

- `Frog`
- `Mint`
- `Ember`
- `Ocean`
- `Custom`

Через тему меняются:

- акцентные линии;
- цвета панелей;
- состояние активных элементов;
- общий визуальный стиль HUD и внутренних окон.

### Hard Accent

Отдельный «критический» цвет, настраиваемый в категории `Theme` тремя ползунками (`Hard accent red / green / blue`). Он не зависит от выбранного пресета и остаётся ярким на любой теме. Hard Accent применяется ровно к двум местам:

- таймер босса, ушедший «в минус» (уже возродившийся босс);
- полоса HP игрока, опустившаяся ниже порога критического здоровья.

## Профайлер

В FrogHelper встроен собственный семплирующий профайлер. Он нужен, чтобы **точно** понять, какая часть мода нагружает клиент, а не гадать — именно через него в несколько итераций оптимизировалась подсветка мира. Это в первую очередь инструмент для диагностики просадок FPS и фризов.

### Ключевые свойства

- **Выключен по умолчанию** и включается вручную. Пока он выключен, замеры не собираются, а сами точки измерения практически ничего не стоят — на обычную игру профайлер не влияет.
- Полностью **клиентский и локальный**: ничего не отправляется на сервер, отчёты сохраняются в файл на вашем диске.
- Замеряет именованные участки кода (секции) и счётчики событий по всему моду: рендер, HUD, отдельные виджеты, тики, обработку протокола и т. д.

### Команда `/fhprof`

| Команда | Что делает |
| --- | --- |
| `/fhprof status` | текущее состояние (вкл/выкл, число секций и счётчиков, длительность сессии) |
| `/fhprof start` | сбрасывает статистику и начинает сбор |
| `/fhprof stop` | останавливает сбор |
| `/fhprof reset` | очищает накопленную статистику |
| `/fhprof report [prefix]` | выводит топ секций и счётчиков прямо в чат |
| `/fhprof dump [prefix]` | сохраняет полный Markdown-отчёт в файл |

Без аргументов `/fhprof` эквивалентен `/fhprof status`.

Типичный сценарий: `/fhprof start` → поиграть в проблемной ситуации 20–60 секунд → `/fhprof dump` → открыть сохранённый файл.

### Фильтр по префиксу

Секции именуются с префиксами: `render/`, `hud/`, `widget/`, `tick/`, `ui/`, `world/`, `protocol/`. Необязательный `prefix` в `report` и `dump` оставляет только совпадающие секции — удобно сфокусироваться на подсистеме:

```text
/fhprof report render/
/fhprof dump render/UsefulWorldHighlightRenderHook
```

### Куда сохраняется отчёт

`/fhprof dump` пишет файл в папку `froghelper-profiler` внутри директории игры:

```text
<game>/froghelper-profiler/fhprof-<дата-время>.md
```

При использовании фильтра имя файла содержит префикс: `fhprof-<prefix>-<дата-время>.md`. Путь к сохранённому файлу выводится в чат.

### Что внутри отчёта

Markdown-отчёт удобно читать/прикладывать при разборе проблем. В нём:

- **Session Context** — версии FrogHelper / Minecraft / Fabric Loader, окружение, сервер, экран, измерение, ник, FPS, размер окна;
- **Summary** — главные горячие точки (по суммарному времени, по self-времени и самый большой одиночный спайк);
- таблицы **Top Sections** — по суммарному времени, по self-времени и по максимальным спайкам;
- **Call Tree** — дерево вложенных замеров;
- **Recent / Worst Samples** — история недавних и худших кадров/тиков;
- **Top Counters** — счётчики событий;
- **Legend** — расшифровка метрик.

### Метрики

| Метрика | Смысл |
| --- | --- |
| `calls` | сколько раз секция была измерена |
| `total` | суммарное время в секции |
| `self` | время в секции без вложенных измеренных участков |
| `avg` | среднее время на вызов |
| `max` | худший одиночный спайк |
| `share` | доля секции от всего измеренного времени |
| счётчики (`events` / `total`) | число событий и накопленные «единицы работы» — это не время, а количество (например, сколько чанков просканировано) |

## Команды мода

### `/fhshare <ник>`

Отправляет ваши локальные таймеры боссов указанному игроку.

### `/fhproto`

Показывает диагностику серверного протокола:

| Команда | Что делает |
| --- | --- |
| `/fhproto` или `/fhproto stats` | выводит текущую статистику пакетов |
| `/fhproto anomalies` | показывает обнаруженные аномалии декодирования |
| `/fhproto reset` | сбрасывает накопленную диагностику |

### `/fhprof`

Управление встроенным профайлером производительности. Подробности и подкоманды — в разделе [Профайлер](#профайлер).
